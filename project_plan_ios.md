# **ShopGuide iOS** 多模态智能导购项目企划书

## 一、项目概述

### 背景

传统电商平台使用图文或短视频进行单向展示，用户如果需求不清晰或有个性化约束，难以从海量商品中做出决策。近年来大模型与检索增强生成（Retrieval‑Augmented Generation，RAG）技术迅猛发展，使得可以通过对话理解用户意图、解析结构化约束，并基于外部知识库给出可信的推荐。本项目旨在构建一个**运行于 iOS 平台的原生多模态电商导购 AI Agent**，完成从自然语言咨询到商品推荐、反选排除、商品对比、购物车管理甚至图片找货的闭环体验。

### 核心目标

1. **全链路闭环**：用户通过对话表达需求，系统能够主动澄清并根据预算、品牌偏好、成分排除等结构化约束推荐商品；支持比较、反选、加购、删除、修改数量以及模拟下单。
2. **约束感知 RAG**：将商品信息以结构化数据和向量方式存储，先解析约束进行结构化过滤，再进行向量检索并生成解释化回复，保证所有价格、库存、优惠信息均来自真实商品库。
3. **多模态查询**：支持用户上传或拍摄图片，利用现有开源项目实现图像特征嵌入和近邻搜索，返回相似品牌的商品卡片。
4. **原生交互体验**：客户端采用 SwiftUI 开发，支持动态浅色/深色模式切换，配色随系统外观自动调整，流式回复与商品卡片嵌入自然，符合苹果人机界面规范。
5. **高工程质量**：模块化后端（FastAPI）、稳定的 SSE 流式接口、可扩展的向量数据库（Chroma/Qdrant），以及完善的目录结构和文档，便于团队协作和后续维护。

## 三、客户端设计

### 3.1 页面和导航结构

* **ChatView**：主聊天界面。顶部 `NavigationStack` 显示标题“智能导购”，右上角放置购物车按钮并显示徽标。中间为消息列表，包括用户气泡、AI 气泡、商品卡片、对比表格和购物车状态卡片；底部为快捷问题 Chips、文本输入框、发送按钮与附件（图片上传）按钮。
* **ProductDetailView**：商品详情页。展示商品大图、名称、价格、品牌、评分、库存、卖点、适用人群等信息，并提供“加入购物车”按钮。
* **CartView**：购物车页。列出已加购商品及其数量和小计金额，支持增减数量、删除、清空及模拟下单，并显示总价。

### 3.2 视图层次结构

* 使用 SwiftUI 的 `NavigationStack` 控制导航，将 ChatView 作为根视图，ProductDetailView 和 CartView 通过导航推入栈中。
* 消息列表采用 `ScrollView` + `LazyVStack`，可纵向滚动并按顺序显示对话记录。
* 商品卡片使用横向 `ScrollView` 展示多件商品，卡片内包含商品图片 (`AsyncImage`)、名称、价格、评分、标签及推荐理由，整体圆角并带阴影。
* 对比表格使用自定义表格视图，按照维度显示商品 A 与商品 B 的比较以及推荐结论。

### 3.3 动态浅色/深色模式设计

苹果的人机界面指南指出，系统提供浅色和深色两种外观，应用应使用动态颜色或在资产目录中为浅色和深色提供不同版本的颜色，这些颜色会根据系统外观自动调整【518906868771330†L264-L305】。因此：

1. **语义颜色**：使用 SwiftUI 提供的 `.primary`、`.secondary`、`.background` 等语义颜色，这些颜色会根据系统外观自动调整对比度，确保浅色模式下文本清晰、深色模式下背景柔和。
2. **品牌色定义**：在 `Assets.xcassets` 中创建自定义颜色 `BrandColor` 和 `AccentColor`，分别提供 Light 和 Dark 两个配色。例如主色在浅色模式下为清新的蓝绿色，而在深色模式下采用更柔和、更高对比度的浅蓝色。通过 `Color("BrandColor")` 使用时，系统会根据当前外观自动加载正确的版本。
3. **阴影与高亮**：深色模式下避免使用纯黑背景，可采用几种不同层级的深灰背景；浅色模式下则采用白色或浅灰色背景。按钮阴影和卡片边框颜色根据模式调整透明度，确保在两种模式下都具有足够的视觉分离。
4. **图标与插图**：使用 SF Symbols 中的可变色图标让系统自动填充合适颜色；自定义图标需要在 Asset Catalog 中提供浅色和深色两个版本。

## 四、图像识别与相似商品搜索功能设计

### 4.1 功能简介

用户可以在聊天界面点击附件按钮上传或拍摄一张商品照片，系统解析照片内容后在商品库中检索相似款式或相同品牌的商品并返回商品卡片列表。该功能属于加分项，展示多模态 AI 在电商导购中的价值。

### 4.2 技术方案

1. **图像特征提取与嵌入**：借鉴 GitHub 开源项目 **SwiftSemanticSearch**，该项目展示了如何使用 **UForm** 模型在 iOS 端实时生成图像与文本的向量嵌入，并通过 **USearch** 执行高效近邻搜索。项目说明指出，该示例应用“使用 UForm 对文本和图片进行多模态嵌入，并使用 USearch 实现实时搜索；同样引擎可扩展至 1 亿条记录”【518906868771330†L264-L305】。本项目可参考其做法：
   * 使用 UForm/CLIP 或其它 CoreML 兼容模型在本地生成上传图片的嵌入向量。
   * 对商品库中的商品主图离线预计算嵌入并存储在向量数据库（如 Qdrant/Chroma）或 USearch 索引中。
2. **近邻搜索**：客户端上传图片后，调用后端的 `/api/image_search` 接口；后端使用同样的模型对图片生成嵌入，利用向量库执行相似度搜索，返回匹配度最高的 N 件商品的 ID、相似度等信息。
3. **结果融合与推荐**：根据搜索相似度分数和商品属性（如库存、价格、品牌）进行排序，并利用大模型生成简短解释。生成内容需要经过 Grounding Guard 校验。
4. **客户端展示**：后端通过 SSE 将图片搜索结果以 `products` 事件返回，前端显示为商品卡片列表，用户可继续咨询、比较或加购。

### 4.3 可选第三方库

* **UForm + USearch**：SwiftSemanticSearch 项目展示了如何使用该组合实现图像与文本语义搜索【518906868771330†L264-L305】。
* **CLIP-Finder**：GitHub 项目 `CLIP-Finder2` 提供了在 iOS 上利用 CLIP 模型进行图片相似搜索的示例，可作为替代方案。
* **SimilaritySearchKit**：Swift 包，提供文本嵌入和向量相似搜索能力，可用于组合图文搜索。

## 五、服务端设计

### 5.1 技术栈

| 层级 | 技术选型 | 说明 |
|---|---|---|
| **Web 框架** | FastAPI | 支持异步请求、SSE 流式响应，开发效率高 |
| **数据库** | SQLite (开发) / PostgreSQL (生产) | 存储商品、购物车、会话等结构化数据 |
| **向量数据库** | Chroma 初始实现，Qdrant 冲高分 | Qdrant 支持向量搜索并通过 payload 过滤执行布尔查询【518906868771330†L264-L305】 |
| **Embedding 模型** | bge-m3 / Doubao-Embedding / UForm | 生成商品文本描述向量 |
| **生成模型** | Doubao-Seed-2.0-lite | 中文语言生成，支持流式输出 |
| **图像模型** | UForm / CLIP / CoreML | 用于图像嵌入和相似搜索 |

### 5.2 API 设计

* `POST /api/chat/stream`：客户端发送用户消息（包含会话 ID 和购物车快照），后端返回 `text/event-stream` 类型的 SSE 流；事件包括 `token`（逐字回复）、`products`（商品列表）、`compare`（比较表）、`cart`（购物车状态）、`error` 和 `done`。
* `GET /api/products/{id}`：根据商品 ID 返回详细信息。
* `POST /api/cart/add` / `remove` / `update`：购物车增删改。
* `POST /api/image_search`：上传图片并返回相似商品列表。

### 5.3 Agent 流程

1. **意图识别**：确定用户的意图类别（推荐、过滤、对比、购物车操作、图像搜索、闲聊）。
2. **约束解析**：解析价格区间、类别、品牌包含/排除、成分包含/排除等结构化信息。
3. **多轮记忆**：维护会话状态，继承上文约束或商品列表，处理指代（如“第一款”）。
4. **工具调用**：根据意图调用产品检索器、比较器、购物车工具或图像搜索工具。
5. **Grounding Guard**：校验生成结果中所有商品是否存在，价格、库存与数据库一致，防止幻觉。
6. **生成回复**：整理检索结果，调用 LLM 生成自然语言，并将文本与结构化数据以 SSE 事件流返回客户端。

### 5.4 数据准备

* **商品表设计**：字段包括 `product_id`、`name`、`category`、`brand`、`price`、`stock`、`image_url`、`rating`、`tags`、`selling_points`、`ingredients`、`skin_type`、`scenario` 等。
* **向量索引**：对商品描述生成向量并写入向量库；对于图片搜索功能，需要对商品图片生成嵌入并保存。
* **数据脚本**：提供 `ingest_products.py` 导入数据至数据库；`build_index.py` 用于批量生成嵌入并构建向量索引。

## 七、阶段实施计划

### 第一阶段：基础能力（1–2 周）

1. 初始化 SwiftUI 和 FastAPI 项目，建立目录结构，确保 iOS 模拟器可访问本地后端。
2. 导入测试商品数据至 SQLite 并创建向量索引。
3. 实现 `POST /api/chat/stream`，集成 Doubao 模型和 SSE；iOS 端实现流式显示和商品卡片。
4. 完成商品卡片组件与详情页；实现购物车页面和基础 API。
5. 在 Asset Catalog 中定义浅色/深色 BrandColor/AccentColor，并全局使用动态颜色。

### 第二阶段：约束与多轮能力（2–3 周）

1. 实现意图分类与约束解析模块，支持多轮对话。
2. 构建会话状态管理，处理指代与约束累积。
3. 支持反选排除和主动澄清，确保结构化过滤准确。
4. 实现商品对比功能并通过 SSE 返回结构化表格。
5. 完善购物车自然语言操作，保持聊天界面和购物车页同步。

### 第三阶段：多模态与优化（2 周）

1. 支持图片上传并在后端生成图像嵌入；建立商品图片的向量索引。
2. 使用 UForm/CLIP 或 USearch 完成图像相似搜索；根据相似度与业务规则排序返回。
3. 完善 SSE 事件类型，添加错误处理和用户提示；优化流式体验。
4. 深化 UI 打磨：使用动画、懒加载、手势操作等提升体验；验证深色/浅色模式的对比度和色彩效果。

### 第四阶段：文档与演示（1 周）

1. 撰写完善的架构、API、RAG、UI、多模态等文档。
2. 制作 Demo 视频，演示从模糊推荐、筛选、反选、对比、购物车、图片找货的完整流程。
3. 进行功能和性能测试，并根据反馈迭代修改，准备答辩。

## 八、风险与对策

1. **模型调用受限**：外部 LLM 可能限流。对策：使用 Doubao-Seed-2.0-lite，自托管 Embedding 模型，并预生成商品嵌入以减少在线计算。
2. **多模态算力不足**：在模拟器或手机上生成图像向量耗时。对策：在服务器端完成嵌入计算，客户端只负责上传；或使用 CoreML 加速实现。
3. **配色适配问题**：深色模式配色不当可能导致对比不足。对策：为浅色和深色分别设计颜色，并通过测试保证符合 WCAG 4.5:1 对比要求；不要简单调整亮度。
4. **隐私与安全**：图片上传涉及用户隐私。对策：在隐私政策中明确说明不存储用户上传的图片；处理完嵌入后立即删除，并加密传输。

## 九、总结

本企划书提出了一个详尽的 iOS 原生多模态电商导购项目方案。核心亮点包括：

* **约束感知 RAG**：通过意图识别、约束抽取和结构化过滤 + 向量检索组合，为用户提供可信赖的商品推荐；所有信息均来自真实商品库，防止模型幻觉。
* **原生流式体验**：SwiftUI 打造现代化界面，支持浅色/深色模式切换，流式回复与商品卡片嵌入提升对话体验。
* **多模态搜索**：参照 SwiftSemanticSearch 等开源项目使用 UForm + USearch 实现图片相似搜索【518906868771330†L264-L305】, 拓展电商导购的交互维度。
* **模块化工程架构**：明确的前后端目录结构和模块划分，配合阶段实施计划，保障项目可扩展性和工程质量。

该方案完全符合挑战赛的要求，在基础功能之上深入多轮对话、图像搜索和暗黑/浅色自适配设计，为打造专业化的电商智能导购应用奠定了坚实基础。
