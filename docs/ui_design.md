# UI 设计说明

## 设计目标

应用定位是工具型电商导购，不做营销落地页。界面应优先服务高频购物决策：看需求、看商品、看依据、选规格、加购。

## 当前界面结构

- 顶部：标题、我的偏好入口、隐私与合规入口和购物车入口。
- 中部：聊天消息、商品卡片、对比卡和购物车状态卡。
- 底部：快捷问题、图片上传按钮、文本输入框、发送按钮。
- 详情页：大图、商品事实、SKU 规格、推荐要点、可信依据、加购按钮。
- 隐私页：解释数据来源、图片处理、偏好记忆、API 使用说明和合规边界。

## 视觉策略

- 使用系统语义色和 Material，支持浅色/深色模式。
- 商品卡使用小圆角、轻阴影和清晰层级，避免粗糙的大块纯灰。
- 聊天气泡和工具栏保留轻量玻璃感，让商品内容成为视觉中心。
- 购物车徽标覆盖在购物车图标右上角，并用系统背景描边隔开底层线条，保证数字可读。
- 深色模式使用系统背景和低饱和青绿色层次，避免大面积纯黑或单一蓝紫色。

## 交互策略

- 多规格商品的卡片按钮显示为规格选择，而不是直接加购。
- 详情页按属性分组选择 SKU，例如颜色、存储、版本。
- 选择不同颜色时展示对应真实 SKU 图片。
- 未采集到真实 SKU 图片时回退主图，不生成伪造图。
- 详情页展示商品来源和可信依据，强化 RAG 可解释性。
- Assistant 空消息使用三点思考动画，避免用户误以为卡住。
- 商品图加载时使用 skeleton 占位，卡片尺寸固定，避免滚动列表跳动。
- 商品卡进入时使用轻量 scale/opacity 动画，购物车徽标使用数字变化动画。
- 模拟下单后自动展示订单后推荐商品卡，帮助用户继续购买配件、补充商品或复购候选。
