# 赛题要求摘要

## 最小闭环

- 原生 iOS 或 Android App，不接受纯 Web/H5。
- 聊天窗口支持文本发送。
- 后端实现 RAG 基本链路并接入向量数据库或向量检索层。
- 使用 SSE 或 WebSocket 返回流式回复。
- 回复中嵌入可点击商品卡片，并可跳转详情页。
- 推荐内容必须来自商品库，不能编造商品、价格、库存或优惠。

## 评分重点

- 基础功能完整性 35%：客户端对话 -> 后端检索 -> 生成 -> 流式返回 -> 商品卡片展示。
- 工程质量 25%：目录清晰、接口合理、错误处理、文档和依赖管理。
- 效果可靠性 20%：检索准确、无幻觉、界面流畅、无明显 Bug。
- 加分项深度 20%：多轮上下文、反选排除、购物车、下单、多模态、性能体验。

## 本项目优先路线

1. 先跑通 FastAPI + SQLite + 本地向量检索 + SSE 的稳定后端。
2. 做 SwiftUI 原生聊天页、商品卡片、详情页、购物车页。
3. 深化多轮上下文、反选排除和对话式购物车，作为主要加分项。
4. 图片找货先保留接口契约，后续接入 CLIP/UForm 图像向量。
